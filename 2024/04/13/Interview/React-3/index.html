  1. react是什么

    1. React 是一个网页 UI框架，通过组件化的方式解决视图层开发复用的问题，本质是一个组件化框架。
    2. 它的核心设计思路有三点，分别是声明式、组件化与通用性。
      1. 声明式的优势在于直观与组合。
      2. 组件化的优势在于视图的拆分与模块复用，可以更容易做到高内聚低耦合
      3. 通用性在于一次学习，随处编写。比如 React Native，React360 等，这里主要靠虚拟 DOM 来保证实现。
    3. 这使得 React 的适用范围变得足够广，无论是 Web、Native、VR，甚至 Shell 应用都可以进行开发。这也是 React 的优势。
    4. 但作为一个视图层的框架，React的劣势也十分明显。它并没有提供完整的一揽子解决方案，在开发大型前端应用时，需要向社区寻找并整合解决方案。虽然一定程度上促进了社区的繁荣，但也为开发者在技术选型和学习适用上造成了一定的成本。

  2. 为什么react要用jsx

    1. react本身并不强制使用JSX 本身可以利用creatElement（’div’,null,hello${this.props.toWhat}）

    2. 什么情况下会触发重新渲染

    3. 渲染报错后怎么处理
      1. 错误边界
        1. getDerivedStateFromError（error）
          1. 更新state 使下一次渲染能够显示降级后的ui

    4. 问题解答：
      1. 问题本质
        1. 在不恰当时机调用了不合适的代码
        2. 在需要调用时，忘记了调用
      2. 解答

        4. shouldComponentUpdate通过返回值true/false 性能优化

        6. 卸载阶段取消定时器

  4. 类组件和函数组件有什么区别

    2. 区别
      1. 语法和写法：类组件是使用类的语法进行定义的，函数组件是使用函数的语法进行定义的。
      2. 状态：类组件具有内部状态，可以通过 setState 来更新；函数组件是无状态的，它们的输出完全由其输入 props 决定。
      3. 复杂度：函数组件更简洁，因为它们只是纯函数；类组件则更复杂，因为它们具有更多的属性和方法。
      4. 生命周期：类组件有生命周期，函数组件没有生命周期，而是使用 Hooks。
      5. 组件调用：类组件实例化后调用 render 方法调用；函数组件只需调用函数即可创建组件。
