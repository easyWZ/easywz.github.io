  1. 对Promise的理解
    1. promise不仅是一个技术工具，更代表了一种编程理念的体现，promise作为一个构造函数，它可以生成具有特定状态的实例，这些状态包括pending（待定）、resolved（成功）和rejected（拒绝），这种状态管理机制，使得异步操作的结果变得可预测和可控。
    2. promise即承诺，后续必要兑现，一旦兑现则不可更改，所以Promise的状态只可更改一次
    3. promise是对回调函数的一种封装，它改进了异步编程的方式。传统的异步编程往往依赖于回调函数，然而回调函数的嵌套过深，很容易导致代码结构混乱，形成所谓的回调地狱问题。而promise则允许我们以同步的方式，表达异步操作，从而可以简化代码臃肿以及可读性差的问题
    4. promise实例可以视为一个状态的展示器，我们可以利用promise来管理具有状态改变的业务逻辑，结合async function进一步提升程序的清晰度和便捷性
    5. promise在前端技术体系中，有着广泛的应用，比如Axios、Fetch API等等，都采用了promise对象来处理异步操作。
    6. 当项目中存在着多个异步操作，并且这些操作之间存在着依赖关系时，promise可以通过链式调用来处理这些操作，使得代码更加简洁和易读，比如用户信息调用数据
    7. 不能滥用，promise.all，如图片的预加载，别嵌套太多。
  2. 延迟加载JS有哪些方式
    1. 动态创建
    2. setTimeout
    3. async
      1. 在外部js加载完成后，浏览器空闲时，Load事件触发前执行
    4. defer
      1. 在js加载完成后，整个文档解析完成后执行
    5. async和defer区别
      1. 下载都是异步的（相较于html解析）
      2. async不考虑依赖，不是顺次序加载； defer在html解析后才执行
  3. JS数据类型有哪些
    1. 基本类型
      1. string
      2. number
      3. boolean
      4. undefined
      5. null
      6. symbol
      7. bigint
    2. 引用类型
      1. object
        1. array
        2. function
        3. date
        4. regExp
    3. 题
      1. true+1 // 2
      2. ‘name’+true // ‘nametrue’
      3. undefined+1 // NaN
      4. typeof null // object
      5. NaN是一个数值类型
  4. null和undefined的区别
    1. 作者先设计的null，借鉴java，根据c语言传统默认转化为0，很容易发生错误
    2. 所以产生了undefined
    3. null是一个表示无的对象（空对象指针），转化为数值时为0；
    4. undefined是一个表示无的原始值，转化为数值时为NaN
  5. ==和===有什么不同
    1. == 比较值 === 比较值和类型
    2. == 比较顺序
      1. 强制null==undefined为true
      2. string==number string转化为number
      3. boolean==number boolean转化为number
      4. object==string||number.. object转化为基本类型
      5. 原理：valueOf返回Math对象的原始值，在后台自动调用，不显示的出现在代码中
  6. 微任务和宏任务
    1. js是单线程语言
    2. js代码执行流程：同步执行完-》事件循环
      1. 事件循环中包含：微任务 宏任务
      2. 同步的任务都执行完了，才会进入事件循环：请求，定时器，事件
    3. for(var i=0;i<3;i++){ setTimeout(function(){console.log(i)},1000*i) } // 333
    4. 微任务：promise.then 宏任务：settimeout
    5. 要执行宏任务的前提是清空了所有微任务
  7. js作用域
    1. 除了函数外，没有块级作用域
    2. 作用域链：内部可以访问外部的变量 外部不能访问内部的变量
      1. 注意：优先查找内部，内部没有才查找外部的
    3. 注意声明变量是用var还是没有写（没写即是window.）
    4. js有变量提升的机制
