微信小程序

本文最后更新于 2025年9月4日 上午

问题

  1. 小程序是什么

答案

  1. 小程序是什么
    1. 最开始用webview 调用js sdk
  2. 小程序优缺点
    1. 只能用wx打开和浏览的H5 js css html wxml
    2. 替换app
    3. 安全
    4. 开发门槛低
    5. 兼容性 autoprefix webview
    6. 流量
    7. 缺点：
      1. 依附于微信
      2. 体积 首次加载最大2M
  3. 小程序生命周期
    1. 应用的生命周期
      1. app.js
      2. App(object)
      3. onLaunch小程序初始化完成时触发，全局只触发一次
      4. onShow小程序启动，或从后台进入前台显示时触发
      5. onHide小程序从前台进入后台时触发
      6. onError小程序发生脚本错误或 API 调用报错时触发
      7. onPageNotFound小程序要打开的页面不存在时触发
      8. onUnhandledRejection()小程序有未处理的 Promise 拒绝时触发
      9. onThemeChange系统切换主题时触发
    2. 页面的生命周期
      1. 打开/离开页面 App(object)
      2. onLoad生命周期回调—监听页面加载发送请求获取数据
      3. onShow生命周期回调—监听页面显示请求数据
      4. onReady生命周期回调—监听页面初次渲染完成获取页面元素（少用）
      5. onHide生命周期回调—监听页面隐藏终止任务，如定时器或者播放音乐
      6. onUnload生命周期回调—监听页面卸载终止任务
    3. 组件的生命周期
      1. 组件自身的函数，在待定的事件 component(object)
      2. created 生命周期回调—监听页面加载
      3. attached 生命周期回调—监听页面显示
      4. ready 生命周期回调—监听页面初次渲染完成
      5. moved 生命周期回调—监听页面隐藏
      6. detached 生命周期回调—监听页面卸载
      7. error 每当组件方法抛出错误时执行
  4. 小程序登陆流程
    1. openid
      1. wx.login()->code->判断是否是已授权的用户->getUserInfo()
      2. 自身的后端调用微信服务器获取用户信息
      3. wx.request 参数（appid appsecret code
      4. 获取到返回的openid session_key
      5. 匹配openid
      6. session_key
      7. session storage
    2. 登陆态是否过期 token
      1. 官方wx.checkSession
  5. 小程序路由跳转
    1. wx.navigateTo 保留当前页面，跳转到应用内的某个页面。但是不能跳到 tabbar 页面
    2. wx.redirectTo 重定向 等价于 wx.close wx.navigateTo 关闭当前页面，跳转到应用内的某个页面。但是不允许跳转到 tabbar 页面
      1. 参数（url success fail complete）
    3. wx.switchTab 跳转至tabBar 关闭所有其他非tabbar的页面
    4. wx.naviagteBack 关闭当前页面，返回上一页面或多级页面。可通过getCurrentPages 获取当前的页面栈，决定需要返回几层
    5. wx.reLaunch 关闭所有页面，打开到应用内的某个页面
  6. 小程序发布流程
    1. 产品组
    2. 设计组
    3. 开发组
    4. 测试组
    5. 提出诉求-设计初稿-开发-体验-测试-发布
    6. 从开发者的角度小程序怎么样发布
      1. 上传代码 devTools dist-commit Version message + bundle
      2. 提交审核 开发管理页面 开发版本-审核提交
      3. 发布版本 灰度版本 线上版本
    7. 发布扩展
      1. 代码管理服务器新拉分枝
      2. 开发 测试
      3. 测试后 本地分枝 合并 master
      4. master最新代码，build
      5. 上传
      6. 审核
      7. 发布
      8. CI CD 自动化 webHooks
      9. 开发提交分支代码
      10. master jenkings
      11. 添加拦截卡口 complier
      12. api push 微信后台 审核
  7. 小程序的支付流程
    1. 用户分享 扫码 进入小程序 选购
    2. 调用微信支付控件
    3. 商家后台也会接收到支付成功
    4. 返回商家小程序 购买成功页
    5. wx支付平台 支付凭证
      1. wx.login code openid
      2. wx pay prepay_id
      3. wx.requestPayment 返回支付结果
  8. 微信小程序实现原理
    1. Hybrid
    2. view webview
    3. jscore
    4. native
    5. https Request 第三方服务器
    6. 双线程的实现机制
    • 小程序通信
    1. render layer
    2. wxml - js
  10. 运行机制
    1. 冷启动 重新开始
    2. 热启动 后台切换到前台
    3. 长期不打开小程序
    4. 短时间内 系统收到2次以上的内存警告
  11. 小程序提升速度
    1. 下载的小程序代码包
    2. 加载小程序代码包
    3. 初始化小程序首页
    4. Appservice ：js引擎初始化 公共代码库注入
    5. webview：公共库的注入 webview 初始化
    6. 加载
      1. 控制小程序包大小
        1. 代码包压缩
        2. 无用的代码 及时清理
        3. 减少包中固定资源的内容 icon assets
        4. 分包加载
    7. 渲染
      1. onload 加载页面
      2. 减少不必要的http请求
      3. list-detail 骨架屏
      4. setData
