一、时间和空间复杂度

时间复杂度

定义

时间复杂度是对算法执行过程中基本操作（例如比较、赋值等）执行次数的量度。它衡量了输入数据规模（通常以 n 表示）增加时，算法运行时间增长的趋势。

常见的表示

  1. 常数时间复杂度O(1)

    1. 无论输入规模多大，算法的运行时间都是恒定的。
    2. 示例：访问数组中的某个元素。

  2. 线性时间复杂度 O(n)

    1. 算法的运行时间随着输入规模的线性增长。
    2. 示例：遍历一个长度为 n 的数组。

  3. 平方时间复杂度 O(n^2)

    1. 运行时间随着输入规模的平方成比例增长，通常出现在嵌套循环中。
    2. 示例：双重循环遍历二维数组。

  4. 对数时间复杂度 O(log n)

    1. 运行时间随着输入规模的对数增加，通常出现在分治算法中，如二分查找。
    2. 示例：二分查找。

  5. 线性对数时间复杂度 O(n log n)

    1. 运行时间是输入规模和对数的乘积，通常出现在高效排序算法中，如归并排序、快速排序
    2. 示例：归并排序。

  6. 指数时间复杂度 O(2^n)

    1. 运行时间随着输入规模的指数倍增长，通常出现在递归算法中，如解决子集问题或斐波那契数列的递归计算。
    2. 示例：斐波那契数列递归计算。

  7. 常见时间复杂度的增长速度比较（从小到大）

    1. O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n)

空间复杂度

定义

空间复杂度是算法在运行过程中占用的内存空间的量度。它衡量了输入数据规模增加时，算法所需的额外存储空间的增长情况。

常见的空间复杂度表示

  1. O(1)：算法所需的额外空间是常量，与输入规模无关。
    1. 示例：只使用常量个变量进行操作的算法。
  2. O(n)：算法所需的额外空间与输入规模成线性关系。
    1. 示例：创建一个新数组来存储与输入规模相同的元素。
  3. O(n^2)：算法所需的空间随着输入规模的平方增长，通常出现在涉及二维数组等数据结构时。

二、基本算法思想

1. 枚举法

  1. 这是最直接、最简单的方法，也被称为“穷举法”。即列举出问题所有可能的解，然后一一验证，直到找到满足条件的解为止
  2. 优点：简单粗暴，几乎能解决所有问题（只要时间允许）。
  3. 缺点：效率通常极低，当问题规模（n）变大时，计算时间会呈爆炸式增长。
  4. 示例：
    1. 百钱百鸡问题：（鸡翁一，值钱五；鸡母一，值钱三；鸡雏三，值钱一；百钱买百鸡，则翁、母、雏各几何？）
      1. 翻译：用100块钱买100只鸡，公鸡5块1只，母鸡3块1只，小鸡1块3只，问各买几只？
      2. 暴力法就是遍历所有公鸡、母鸡、小鸡数量的组合（从0到100），检查哪种组合满足总钱数和总只数的要求。
    2. 密码破解：尝试所有可能的密码组合。

2. 分治法

  1. “分而治之”是解决复杂问题的强大策略。核心思想是：将一个规模为 n 的大问题，分解成 k 个规模较小的子问题，这些子问题相互独立且与原问题形式相同。递归地解决这些子问题，然后将子问题的解合并起来，就得到原问题的解。
  2. 步骤即为：分解 -> 解决子问题 -> 合并。
  3. 示例
    1. 归并排序：将待排序数组一分为二，分别对左右两个子数组进行排序（递归进行），最后将两个已排序的子数组合并成一个有序数组。
    2. 快速排序：选择一个基准元素，将数组分成“比基准小”和“比基准大”的两部分，再对这两部分递归进行快速排序。
    3. 二分查找：在有序数组中，每次都通过与中间元素的比较，将待查找区间缩小一半，直到找到目标或区间为空。

3. 贪心算法

  1. 贪心算法在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择。核心思想即希望通过一系列局部最优选择，从而导致全局最优解。
  2. 优点：效率高，通常时间复杂度低。
  3. 缺点：不是所有问题都能得到全局最优解。它目光短浅，无法回退。
  4. 示例：
    1. 找零钱问题：用最少张数的纸币凑出某个金额。在人民币体系下（有100, 50, 20, 10, 5, 1元），贪心策略是每次都选择不超过剩余金额的最大面额纸币。
    2. 霍夫曼编码：一种用于数据压缩的算法，通过构造最优二叉树，让出现频率高的字符使用更短的编码。

4. 动态规划

  1. 动态规划是解决具有重叠子问题和最优子结构性质问题的利器。
  2. 核心思想：将复杂问题分解为相对简单的子问题，但不同于分治法，动态规划的子问题往往是重叠的。为了避免重复计算，它会保存子问题的解（通常用数组或字典），在需要时直接查找，从而大大提高效率。这是一种“用空间换时间”的思想。
  3. 步骤：
    1. 定义状态：找出问题中的变量，定义状态（通常用一个数组 dp[i] 来表示）。
    2. 建立状态转移方程：找出状态之间的关系式，即如何从已知状态推导出未知状态。
    3. 确定初始条件和边界。
    4. 计算顺序：从小到大或从大到小计算状态。
  4. 示例
    1. 斐波那契数列：F(n) = F(n-1) + F(n-2)。直接递归计算会重复计算大量子问题，而动态规划用一个数组存储已经计算过的 F(i)。
    2. 背包问题：给定一组物品的重量和价值，在限定的总重量内，如何选择物品使得总价值最大。
    3. 最短路径问题。

5. 回溯法

  1. 回溯法是一种通过试错来寻找所有（或部分）解的算法。当它发现某一步骤不能得到有效的解时，它将取消上一步甚至上几步的计算，再通过其他的可能路径继续寻找。
  2. 核心思想：“前进，失败，回溯，再尝试”。它是一种系统性的搜索技术，通常用递归来实现。
  3. 特点：比暴力枚举聪明，因为它会在发现当前路径不可能成功时提前放弃（称为“剪枝”）。
  4. 示例：
    1. 八皇后问题：在8×8的国际象棋棋盘上摆放8个皇后，使其不能互相攻击（即任意两个皇后都不能处于同一行、同一列或同一斜线上）。回溯法会尝试在第一行放一个皇后，然后在第二行安全位置放一个，如此继续。如果某一行没有安全位置，就回溯到上一行，移动皇后的位置再试。
    2. 数独求解。
    3. 全排列问题。
