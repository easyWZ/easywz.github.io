历史

  1. 1991年：Sun 公司启动 “Green Project”，旨在为嵌入式设备（如机顶盒）开发一种新语言，最初命名为 “Oak”（橡树）。
  2. 1994年：受互联网浪潮影响，团队将 Oak 转向 Web 开发，并更名为 “Java”（灵感来自印度尼西亚的爪哇岛咖啡）。
  3. 1995年：Sun 正式发布 Java 1.0，口号是 “Write Once, Run Anywhere”（一次编写，到处运行），凭借跨平台能力（基于 JVM 虚拟机）迅速流行。
  4. 2006年：Sun 开源 Java，成立 OpenJDK 项目。
  5. 2009年：Oracle 收购 Sun，成为 Java 主要维护者。
  6. 2018年：Oracle 调整许可模式，对商用 Java SE 收费，推动 OpenJDK 和第三方免费发行版（如 Amazon Corretto、AdoptOpenJDK）的普及。
  • Android 崛起：Google 采用 Java 作为 Android 应用开发语言（后转向 Kotlin，但 Java 仍广泛使用）。
  • 企业生态：Java 主导企业级开发，Spring、Hibernate、Apache 等项目成为行业标准。
  • 每 6 个月发布一个特性版本，每 2-3 年推出一个 LTS 版本。

概念

JVM,JRE,JDK

JVM

  1. JVM (Java Virtual Machine) — Java虚拟机
  2. JVM 是“虚拟机”，一个抽象的计算机，它是 Java “跨平台”能力的核心
  3. 负责加载 .class 字节码文件；验证字节码的安全性；解释/编译字节码为所在操作系统的本地机器码并执行；管理内存（垃圾回收 GC 就在这里发生）；提供线程管理等功能
  4. JVM 是 “平台相关” 的，Windows 有 Windows 版的 JVM，Linux 有 Linux 版的 JVM。正是这些不同平台上的 JVM，才保证了同一个 .class 文件能在所有系统上运行。
  5. “一次编译，到处运行” 的本质：Java 源码被编译成与任何特定平台无关的 “.class”字节码，这个字节码由各平台的 JVM 来执行。

JRE

  1. JRE (Java Runtime Environment) — Java运行时环境
  2. JRE 是“运行环境”。如果你想运行一个已有的 Java 程序（比如一个用 Jar 包打包的桌面应用），你只需要安装 JRE。
  3. JRE = JVM + 运行所需的核心类库。它是一个运行环境，不包含任何开发工具（如编译器）。

JDK

  1. JDK (Java Development Kit) — Java开发工具包
  2. JDK 是“开发工具包”。如果你想编写、编译、调试、运行 Java 程序，你必须安装 JDK
  3. JDK = JRE + 开发工具

总结

JDK > JRE > JVM
自 Java 9 模块化以后，官方不再单独提供面向公众的 JRE 安装包。通常开发者直接安装 JDK，而对于运行环境，可以通过 JDK 来生成一个定制化的、更小的 JRE（使用 jlink 工具）。所以，现在“安装 Java”通常默认就是指安装 JDK。

Java SE、Java EE、Java ME

  1. Java SE (Standard Edition) - 标准版: Java 技术的核心和基础, 所有其他版本都以其为起点构建或裁剪
    1. 用途：开发桌面应用程序（如 Eclipse IDE）、命令行工具、简单的服务器应用。学习 Java 编程，就是从学习 Java SE 开始。
    2. 我们常说的 “Java 8”、 “Java 17” 指的就是 Java SE 的版本
  2. Java EE (Enterprise Edition) - 企业版: 基于 Java SE 之上的一系列“规范”和“API”的集合，用于简化大规模、多层、分布式企业应用的开发。
    1. 2017年，Oracle 将 Java EE 的管理权移交给了 Eclipse 基金会。
    2. 由于商标问题，Java EE 已正式更名为 Jakarta EE。现在我们说企业级 Java，通常指 Jakarta EE。
    3. 用途：开发电子商务平台、客户关系管理系统、银行交易后端等复杂系统。
    4. 虽然 Jakarta EE 是官方标准，但在实际开发中，Spring Framework（及 Spring Boot）因其更轻量、灵活、易用的特点，已成为构建 Java 企业应用的事实标准。Spring 大量借鉴并实现了 Java EE 的思想。
  3. Java ME (Micro Edition) - 微型版
    1. Java SE 的一个功能子集，专门为内存、处理器和存储空间极其有限的设备优化
    2. 用途：在智能手机时代之前，广泛用于功能手机游戏、蓝光播放器菜单、智能卡、工业传感器等嵌入式系统。
    3. 随着 Android（基于Java语法，但有自己的SDK和运行时）和 iOS 的崛起，Java ME 在移动设备市场已基本退出历史舞台。目前主要在一些特定的嵌入式物联网领域仍有应用，但影响力已大不如前。

安装

  1. JDK 安装包下载：
    官网下载最新的稳定版JDK
  2. 安装JDK
  3. 查找本地java安装路径（macOS）执行命令 /usr/libexec/java_home 显示 /Library/Java/JavaVirtualMachines/jdk-21.jdk/Contents/Home
  4. 查看所有已安装版本：执行命令 /usr/libexec/java_home -V 会列出所有已安装JDK的版本号和对应路径
  5. vim ~/.zshrc 添加环境变量
  6. source ~/.zshrc更新文件
  7. java -version 查看本地安装版本结果
