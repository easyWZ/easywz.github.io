1. Docker

Docker 是一个开源的「容器化」平台，可以把你的应用和它所需要的环境（代码、依赖、系统工具等）一起打包成一个“集装箱”。这个箱子可以在任何安装 Docker 的机器上一键运行，完全不受底层操作系统差异的影响。

Docker 能干什么？

  1. 统一开发环境:
  • 团队里有人用 macOS，有人用 Windows，有人用 Linux。写一个 Dockerfile + docker-compose.yml，所有人运行同样的命令就能获得完全一致的环境，再也不会出现“在我电脑上是好的啊”。
  1. 简化应用部署:
  • 传统部署需要手动安装 Node、Nginx、数据库，配置环境变量……Docker 只需：
  • 镜像可以直接推送到仓库，在服务器上一键拉取运行。
  1. 微服务架构:
  • 每个服务（前端、后端、数据库、缓存）分别跑在自己的容器里，互相隔离又可以通过网络通信。用 docker-compose 可以一键启动整套系统。
  1. CI/CD 流水线:
  • 在 GitHub Actions、GitLab CI 等平台中，可以直接使用 Docker 容器作为构建环境。每次代码提交后自动创建干净的环境进行测试、打包，保证结果可重现。
  1. 本地软件测试:
  • 想试试 Nginx 最新版？docker run -p 8080:80 nginx。想跑一个 Redis？docker run -p 6379:6379 redis。用完直接删除容器，不留任何垃圾。
  1. 数据库临时环境:
  • 想跑一个 MySQL 数据库？docker run -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -d mysql:5.7。想跑一个 PostgreSQL 数据库？docker run -p 5432:5432 -e POSTGRES_PASSWORD=123456 -d postgres:14-alpine。用完直接删除容器docker rm -f，不留任何垃圾。

2. Docker的安装

  1. 下载Docker安装包
  2. 配置国内镜像加速器:
    1. 点击屏幕顶部的 Docker 图标，在菜单中选择 "Settings" (设置)。
    2. 在左侧菜单栏中找到 "Docker Engine" 并点击。
    3. 在右侧的JSON配置框中，找到 "registry-mirrors" 这一项。如果它不存在，就手动添加。
      1. 2026年可用的国内加速源列表:
      • https://docker.m.daocloud.io
      • https://dockerproxy.com (备选)
      • https://docker.nju.edu.cn (南京大学镜像站)
      • https://docker.mirrors.ustc.edu.cn (中科大镜像站)
    4. 配置好之后，点击界面右下角的 "Apply & Restart" (应用并重启) 按钮。

  2. 在浏览器中访问 http://localhost:8080

  2. 有了项目后使用 Docker Compose 启动开发环境。
    docker compose watch

  3. 安装完 在浏览器中打开 http://localhost

  4. 环境里有什么？

现在环境已经启动并运行，里面实际上有什么？从高层次来看，有几个容器（或进程），每个容器都为应用程序提供特定的功能：

  • React 前端 - 一个运行 React 开发服务器的 Node 容器，使用 Vite。
  • Node后端 - 后端提供了一个API，该API提供了检索、创建和删除待办事项的能力。
  • MySQL数据库 - 用于存储项目列表的数据库。
  • phpMyAdmin - 一个用于与数据库交互的基于Web的界面，可通过 http://db.localhost 访问。
  • Traefik 代理 - Traefik 是一个应用程序代理，可以将请求路由到正确的服务。它将所有针对 localhost/api/* 的请求发送到后端，将针对 localhost/* 的请求发送到前端，然后将针对 db.localhost 的请求发送到 phpMyAdmin。这使得可以使用端口 80 访问所有应用程序（而不是为每个服务使用不同的端口）。

在这个环境中，作为开发者的你无需安装或配置任何服务，无需填充数据库架构，无需配置数据库凭据，也无需其他操作。你只需要 Docker Desktop。其余的一切都会自动运行。

5.构建并推送第一个镜像

什么是容器镜像

  1. 容器镜像（Container Image）是用于创建容器的模板。每个容器镜像都有一个唯一的标识符，例如 nginx:1.25-alpine。镜像包含容器运行所需的所有文件和配置，以及启动容器的指令。

  2. 容器镜像，可以将它们视为一种标准化的软件包，其中包含了运行应用程序所需的一切，包括其文件、配置和依赖项。这些软件包随后可以被分发并与他人共享。

Docker Hub

Docker Hub 是一个免费的容器镜像仓库，用于存储、分发和共享容器镜像。

上述demo项目中，就使用了以下来自 Docker Hub 的镜像，每个镜像都是 Docker 官方镜像：

  • node:18-alpine - 提供了一个 Node 环境，并作为您开发工作的基础。此镜像也用作最终应用程序镜像的基础。
  • mysql:5.7 - 提供一个 MySQL 数据库来存储待办事项列表项
  • phpmyadmin/phpmyadmin - 提供 phpMyAdmin，一个基于 Web 的 MySQL 数据库管理界面
  • traefik:2.9 - 提供 Traefik，这是一个现代的 HTTP 反向代理和负载均衡器，根据路由规则将请求路由到适当的容器
  • docker/welcome-to-docker - 提供一个欢迎页面，用于介绍 Docker 和 Docker Compose
