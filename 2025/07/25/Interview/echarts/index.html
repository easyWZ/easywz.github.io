1. Charts 是什么？用在哪些场景？

ECharts 是百度开源的 JavaScript 图表库，基于 Canvas 渲染，支持丰富的图表类型（柱状图、折线图、饼图、地图、雷达等）。常用于数据可视化、运营大屏、BI 报表、仪表盘等。

2. 使用 ECharts 的基本步骤？

1. 引入 ECharts（CDN 或 npm 安装）
2. 准备一个 DOM 容器（宽高不能为 0）
3. 使用 echarts.init(dom) 创建实例
4. 使用 setOption(option) 设置图表配置
5. 根据需要绑定事件，如点击、缩放、resize 等

3. ECharts 是基于 Canvas 还是 SVG？

默认使用 Canvas 渲染，性能高；但也支持使用 SVG 渲染器（v5 起），适用于交互性较强、图形编辑场景。
echarts.init(dom, null, { renderer: 'svg' })

4. 如何实现图表的自适应宽高？

•	容器的宽高应设置为百分比或响应式
•	在窗口大小变化时，调用 chart.resize() 方法即可

5. 如何实现 tooltip 联动？

使用 tooltip.trigger: ‘axis’ 可使多个系列在同一轴线上联动显示 tooltip。

8.大量数据下如何优化折线图性能？

•	启用 sampling: 'average'（采样降低数据点）
•	使用 large: true, largeThreshold: 10000
•	减少阴影、动画等开销
•	开启 dataZoom 滚动加载

9. ECharts 图表数据更新的方式？

•	可以使用 setOption(newOption, true) 强制更新
•	或者只更新部分数据（不销毁实例）

10. 如何销毁 ECharts 实例？

chart.dispose()
一般用于组件卸载、内存释放。

11. ECharts 底层渲染原理？

ECharts 使用 ZRender 图形库做底层绘图，默认基于 Canvas 实现图形渲染，每次 setOption 会重绘整个 Canvas 图层，图层结构分为多个独立层管理，使用 requestAnimationFrame 实现异步绘制。

12. ECharts 与 Vue/React 等框架怎么集成？

•	可以封装为组件
•	使用 ref 引用 DOM 创建实例
•	利用 onMounted、useEffect 等生命周期初始化、resize、更新

13. 图表刷新频率高如何优化？

•	使用 throttle 限流 resize 或数据刷新频率
•	更新数据时避免全量 setOption
•	数据量大时使用懒加载分页、滚动加载

14. ECharts 如何处理多个图表联动？

使用官方提供的 echarts.connect() 实现多个图表联动缩放、tooltip 等。
echarts.connect([chart1, chart2])

15. ECharts 是否支持服务端渲染（SSR）？

默认不支持 SSR，因为依赖 window 和 DOM，但可以通过 echarts-gl + puppeteer 在服务器上渲染图表并生成图片导出。

16. 如何实现地图中省份点击跳转详情页？

•	使用 geo 或 map 组件注册中国地图
•	绑定 click 事件，根据 params.name 获取点击区域

17.如何实现动态数据更新的仪表盘？

•	使用 setInterval 更新数据
•	在 setOption 中替换 series[0].data 即可
