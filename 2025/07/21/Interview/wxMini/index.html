1. 小程序的运行机制是怎样的？

小程序采用逻辑层（JS）+ 视图层（WXML/WXSS）分离的架构，双线程通信：
• 逻辑层运行在 JavaScriptCore，处理数据逻辑；
• 视图层使用 WebView 渲染 UI；
• 二者通过 微信的原生桥（Native Bridge）通信。

2. 小程序的页面生命周期函数有哪些？

onLoad
页面加载
onShow
页面显示
onReady
页面初次渲染完成
onHide
页面隐藏
onUnload
页面卸载
组件还有如 created, attached, ready, detached 等。

3. 小程序中 data 中的数据会实时渲染到视图吗？

只有使用 this.setData() 修改的数据才会同步到视图层，直接修改 this.data.xxx = ‘xxx’ 不会更新视图。

4. WXML 与 HTML 有什么区别？

•	WXML 是小程序专用模板语言，不支持 DOM 事件绑定如 onclick；
•	数据绑定采用双大括号 {{}}；
•	标签为小程序专用组件，如 <view>、<text> 等；
•	不能直接使用 <div>、<span>、<input> 等原生 HTML 元素。

5. 小程序中如何进行页面跳转？

方法
是否保留当前页面

wx.navigateTo
是（最多10层）
wx.redirectTo
否
wx.switchTab
用于跳转到 tab 页面
wx.reLaunch
关闭所有页面并打开新页面
wx.navigateBack
返回上一级

6. 小程序组件之间如何通信？

•	父传子：使用 properties 传参；
•	子传父：this.triggerEvent('自定义事件', data)
•	非父子通信：
•	使用全局事件总线（自建）；
•	使用全局状态管理（如 MobX、Redux 小程序适配版）；
•	利用 Storage 临时共享数据。

7. 自定义组件如何封装？

8. 如何引入第三方组件库（如 Vant Weapp）？

1. 通过 npm 安装并构建；
2. 或者下载源码放入项目；
3. 在页面 json 中注册组件路径；
4. 在 wxml 中使用对应标签。

9. 微信小程序登录授权流程

wx.login() 获取 code
↓
后端使用 code 调 jscode2session
↓
获取 openid / session_key
↓
生成并返回 token
↓
小程序存储 token，后续请求使用
