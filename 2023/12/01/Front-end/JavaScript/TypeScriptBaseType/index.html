一、什么是 TypeScript？

  1. TypeScript 是 JavaScript 的一个超集（superset）
    1. 所有合法的 JavaScript 代码，同时也是合法的 TypeScript 代码。
    2. 它在 JavaScript 的基础上添加了静态类型系统和其他一些面向对象编程的特性（如接口、泛型、枚举等）。
    3. 通俗理解：TypeScript = JavaScript + 类型检查 + 增强的工具支持。

二、为什么需要 TypeScript？

  1. JavaScript 是动态类型语言，你在编写时不需要声明变量是什么类型。这在小型项目中很灵活，但到了大型项目，动态类型会带来几个问题：
    1. 运行时才发现的类型错误：比如你期望 user.name 是字符串，但某处改成了数字。
    2. 函数参数和返回值不清晰：看函数签名很难知道该传什么、返回什么。
    3. IDE 支持弱：代码提示、重构、重命名等功能受限。
  2. TypeScript 通过添加 静态类型系统（在代码运行前检查类型错误）来解决这些问题。

三、如何运行 TypeScript？

  1. TypeScript 不能在浏览器或 Node.js 中直接运行。你需要先用 TypeScript 编译器（tsc）将 .ts 文件 编译（转译）成纯 JavaScript 文件，然后再运行 JS。依照下述步骤：
    1. npm install -g typescript
    2. 使用.ts扩展名创建TypeScript文件 例greeter.ts
    3. 编译代码 tsc greeter.ts 输出结果为一个 greeter.js文件
  2. 类型擦除 ： TypeScript 在编译成 JavaScript 时，把所有的类型注解、接口、类型别名等全部“擦掉”，只留下纯 JS 代码。

七、 函数重载

函数重载是指同一个函数名，根据不同的参数类型、个数或顺序，可以有不同的行为或返回值类型。在 TypeScript 中，它让你为一个函数定义多个类型签名，但底层只实现一次。

JavaScript：本身不支持重载。后面的同名函数会覆盖前面的。
TypeScript：支持重载的类型声明。你可以写多个“重载签名” + 一个“实现签名”。

应用场景：

  1. createDate 可接受时间戳或年月日 createDate(1609459200000) ; createDate(2021, 0, 1)
  2. getElement 不传 id 返回所有元素，传 id 返回单个 getElements() → Array<Element>; getElements('#id') → Element
  3. fetch 的返回值类型随参数变化 fetch('user') → User; fetch('post') → Post

对象类型工具

主要用来修改或转换对象的结构。

类型 作用 描述
Partial<T> 把 T 的所有属性变成可选 适合做“更新”操作：我只想更新对象一部分数据，其他的保留
Required<T> 把 T 的所有属性变成必填 适合做“创建”操作：我必须提供对象的所有属性
Pick<T, K> 从 T 中选择 K 类型的属性 适合做“投影”操作：我只想选择对象的一部分属性
Omit<T, K> 从 T 中排除 K 类型的属性 适合做“投影”操作：我只想排除对象的一部分属性
Readonly<T> 把 T 的所有属性变成只读 用于定义一些“只读配置”，防止被意外修改
Record<K, T> 创建一个对象类型，其键名是 K，键值是 T 适合用来定义“字典结构”，比如 Record<string, User> 就是一个键为字符串、值为 User 类型的对象

集合/联合类型工具

主要用于“筛选”和“组合”联合类型

类型 作用 描述
Exclude<T, U> 从 T 中排除 U 类型的元素 适合做“投影”操作：我只想保留 T 中不是 U 类型的元素
Extract<T, U> 从 T 中提取 U 类型的元素 适合做“投影”操作：我只想保留 T 中是 U 类型的元素
NonNullable<T> 从 T 中剔除 null 和 undefined 确保你的类型“不是空值”
ReturnType<T> 从 T 中提取返回值类型 用于类型推断，获取某个函数的返回结果类型。
Parameters<T> 从 T 中提取参数类型 用于类型推断，获取某个函数的参数类型。
InstanceType<T> 从 T 中提取实例类型 用于类型推断，获取某个类的实例类型。

九、 any 类型

为什么需要 any？

  1. 渐进式迁移：从 JS 到 TS 的桥梁

    1. TypeScript 最大的成功之一就是渐进式。当你把一个大型 JavaScript 项目迁移到 TypeScript 时，不可能一夜之间为所有代码加上完美的类型。any 让你可以：
      1. 临时忽略类型错误，继续迁移代码。
      2. 逐步为代码添加类型，确保类型安全。
    2. 如果没有 any，迁移成本会高得可怕，许多人会放弃使用 TypeScript。

  2. 有些场景在编译时确实无法知道类型，比如第三方无类型库，动态 API 响应，反射、元编程

  3. 极少数情况下，类型系统不足以表达某些高级模式（如递归类型、复杂类型变换）。any 可以作为临时解决方案。

如何避免滥用 any？

  1. 启用严格模式（strict: true）
  2. 使用 ESLint 规则
  3. 用 unknown 代替 any,使用类型断言（as）或类型守卫
    1. unknown 是类型安全的“any” —— 它表示“我不知道是什么类型”，但在使用前必须收窄类型：
      1. 你可以使用类型断言（as）来指定窄类型。
      2. 你可以使用类型守卫（if (typeof user === 'string')）来检查类型。

题目

10. Promise 封装考查

题目

更高难度附加练习：创建一个 promisifyAll 函数，接受一个包含函数的对象，返回一个新对象，其中每个函数都被 promisified。

答案

改动说明：

  1. 定义 CallbackFunction 类型：CallbackFunction<T> 表示接受回调函数的函数类型，回调参数为 ApiResponse<T>
  2. 实现 promisify 函数：
    • 接受一个回调风格的函数
    • 返回一个新函数，该函数返回 Promise
    • 在 Promise 内部调用原函数，成功时 resolve data，失败时 reject Error
  3. 实现 promisifyAll 函数（Bonus）：
    • 使用映射类型遍历对象的每个属性

    • 使用条件类型和 infer 提取返回类型
  4. 更新 api 创建：使用 promisifyAll(oldApi) 替代逐个 promisify
  5. 类型安全：所有 promisified 函数返回正确的 Promise 类型
